# 项目规范

## 联网抓取策略

当需要抓取网页信息时，按以下优先级选择工具：

1. **首选 `WebFetch`** — 适用于已知 URL、需要提取页面特定信息的场景
2. **若 `WebFetch` 失败**（如网络限制、反爬拦截、页面需要 JS 渲染、内容不完整等），**升级为 CDP 模式抓取**：
   - 加载 `web-access` skill
   - 通过浏览器 CDP 直连用户日常浏览器（Chrome/Edge），天然携带登录态
   - 使用 `/new` 创建后台标签页，`/eval` 提取内容，`/screenshot` 截图采帧
   - 任务完成后用 `/close` 关闭自建标签页，保留用户原有标签页

### 判断升级 CDP 的信号
- `WebFetch` 返回错误、超时或空内容
- 目标页面是动态渲染（SPA、小红书、微信公众号等反爬严格的平台）
- 页面需要登录态才能获取完整内容
- 需要与页面交互（点击、滚动、填表）才能暴露目标信息
- 需要获取视频帧、图片资源等富媒体内容

### CDP 使用原则
- 所有操作在自建后台标签页中进行，不操作用户已有标签页
- 优先尝试程序化操作（构造 URL、`/eval` 操作 DOM），受阻时切换 GUI 交互（`/click`、`/scroll`）
- 提取 URL 时保留完整地址（含会话参数），不裁剪或省略
- 打开页面后先尝试获取目标内容，确认无法获取且判断登录能解决时，再提示用户登录
